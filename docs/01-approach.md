# 01. 접근 방식 (Approach)

> **요약**: M.O.L.E은 공식 한국어를 지원하지 않는다. 우리는 게임에 새 `ko` 로케일을 추가하는 대신, **사용되지 않는 일본어(ja) 로케일 슬롯을 한국어로 덮어쓰는(하이재킹)** 가장 미니멀한 전략을 택했다. 사용자는 게임에서 언어를 `日本語`로 선택하면 한국어를 보게 된다.

## 문제 정의

- **대상**: Steam 게임 *M.O.L.E* (개발: Z Labs). 심리 호러. Unreal Engine 5.
- **현황**: 상점 페이지에는 한국어 설명이 있으나 **인게임 한국어 지원은 없음.**
  게임이 패키징한 지원 언어:
  `en, fr, ja, zh-Hans(중국어 간체), de, es, pl, pt, ru, uk` — **`ko` 없음.**
- **작업 환경**: WSL(Ubuntu) 안에서 Windows Steam 설치 경로에 접근.
  `/mnt/c/Program Files (x86)/Steam/steamapps/common/M.O.L.E/...`
  WSL은 Windows 실행파일도 호출 가능하고 인터넷·Python도 사용 가능 → 분석·빌드를 WSL 안에서 수행.

## 큰 그림: 일본어 슬롯 하이재킹

핵심 아이디어는 **새 언어를 추가하지 않고, 안 쓰는 언어 한 칸을 한국어로 바꾸는 것**이다.

1. 게임의 일본어 번역 파일(`.../Localization/Game/ja/Game.locres`)의 텍스트를 한국어로 교체한다.
2. 게임의 한글 글리프가 없는 폰트들을, 한글을 포함한 도트 폰트로 교체한다.
3. 위 변경을 별도 패치 pak(`Mole-Windows_P.pak`)으로 묶어 게임 폴더에 추가한다.
4. 사용자는 게임 설정에서 언어를 **`日本語`**로 선택한다 → 화면에 한국어가 나온다.

### 왜 일본어(ja)인가
- **거의 안 쓰는 슬롯**: 한국 사용자에게 일본어는 불필요 → 덮어써도 손실이 없다.
- **구조가 동일**: 모든 언어의 `Game.locres`는 같은 키 집합/구조를 공유한다. ja 슬롯만 바꾸면
  나머지 언어는 그대로 동작한다.
- **폰트 호환**: 일본어 폰트는 한자(漢字)를 포함하므로, 한글+한자를 모두 가진 폰트로 교체하면
  전환기에 미번역 일본어가 남아 있어도 깨지지 않는다([03](03-fonts.md) 참조).

### 대안: 새 `ko` 로케일 추가 — 왜 안 했나
새 한국어 로케일을 "정식으로" 추가하려면:
- 게임의 **언어 선택 목록 UI**에 항목 추가,
- 엔진 **컬처(culture) → 로케일 매핑**,
- 폰트 **컬처별 매핑(컴포지트 폰트)** 수정,
- 신규 `ko/Game.locres`/`Game.locmeta` 생성 및 등록

까지 모두 손대야 한다. 이는 IoStore 안의 블루프린트/위젯/설정 에셋을 수정해야 하므로 훨씬
복잡하고 게임 업데이트에 취약하다. **하이재킹은 단 두 종류의 파일(locres·폰트)만 교체**하면
되므로 가장 미니멀하고 견고하다.

## 결과 요약

- **본문 번역**: `Game.locres` 1300개 문자열 전부 한국어화 (≈ 33,700자).
- **엔진 키 이름**: `Engine.locres`에서 인게임에 표시되는 키 표시명 115개 한국어화.
- **폰트**: 커스텀 폰트 6종을 모두 Galmuri11(한글+영문+가나+한자)로 교체.
- **산출물**: 단일 드롭인 파일 `Mole-Windows_P.pak`. 원본을 덮어쓰지 않고 추가만 하며,
  파일 삭제만으로 제거 가능.
- **재현성**: 게임 원본 에셋 없이 `data/` + `fonts/` + 사용자 게임만으로 패치를 재빌드
  (`tools/build.py`). 빌드는 **결정적**이라 같은 게임 버전·폰트면 배포본과 **바이트 단위로 동일**(repak을 단일 스레드로 호출해 보장).
