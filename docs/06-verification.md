# 06. 전수 검증 — "빠진 텍스트는 없는가?"

> 번역·폰트·키 이름을 다 처리한 뒤, "혹시 게임 어딘가에 우리가 손대지 못한 일본어 텍스트가 하드코딩돼 있지 않은가?"를 정면으로 검증했다. 결론은 **없다** — 모든 표시 텍스트는 `Game.locres`를 거치며, IoStore에 박힌 일본어 UI 텍스트는 0건이다.

관련 문서: [02. 리버스 엔지니어링](02-reverse-engineering.md) · [05. 엔진 키/버그](05-engine-keys-and-bugs.md)

---

## 1. 검증 논리

핵심은 **게임의 소스 컬처가 영어**라는 점이다. 언리얼은 쿠킹 시 소스 언어 문자열을 에셋(uasset) 안에 그대로 박고, 번역(일본어·한국어 등)은 `*.locres`에만 둔다. 따라서:

- uasset 안에서 **영어 원문**이 나오는 것은 정상이다.
- uasset 안에서 **일본어 게임 단어**가 나온다면, 그것은 locres를 거치지 않는 **하드코딩 텍스트** = 우리가 번역으로 못 고치는 누락이다.

그래서 "uasset 안에 실제 일본어 단어가 있는가?"만 확인하면 된다.

---

## 2. IoStore 압축 해제

게임 본문 에셋은 IoStore 컨테이너(`Mole-Windows.ucas`, 857MB, Oodle 압축)에 들어 있다. 압축 상태에서는 `strings` 검색이 무의미하므로 전부 풀었다.

```bash
retoc unpack Mole-Windows.utoc /tmp/ucas_unpack
# → unpacked 10,870 files (3.1GB)
```

---

## 3. 함정 1 — `strings` 는 일본어를 건너뛴다

처음엔 `strings -e l`(UTF-16LE)로 가나를 찾으려 했지만 **0건**이 나왔다. 원인은 `strings`가 일본어 문자를 "비출력(non-printable)"으로 간주해 **출력에서 빼버리기** 때문이다. 즉 0건은 "없음"이 아니라 "도구가 못 봄"이었다.

→ 도구를 믿지 말고 **바이트 단위 정규식**으로 직접 스캔했다.

- UTF-16LE 가나: 상위 바이트가 `0x30`인 코드유닛(히라가나 U+3040–309F, 가타카나 U+30A0–30FF)
- UTF-8 가나: `E3 81–83` 시퀀스

---

## 4. 함정 2 — 오탐(메시·텍스처·신경망 바이너리)

2글자 이상 연속 가나로 스캔하니 601개 파일이 잡혔다. 그러나 전부 **오탐**이었다.

```
WBP_Links.uasset   : ['メが']
BP_Monster.uasset  : ['たた', 'メが']
SM_GoodSky.uasset  : ['ョョ', 'ネネ']
```

`メが`, `ョョ`, `ゲゲ` 같은 2글자 조각은 의미 없는 바이트 패턴이다(모든 BP에 공통으로 나오는 `メが`는 블루프린트 바이트코드).

임계값을 4글자로 올리니 더 명확해졌다 — 잡힌 것은 전부 `SM_`(스태틱 메시), `T_`(텍스처), `NNE_`(신경망 가중치), `MERGED_StaticMeshActor`였고, 내용은:

```
ぁあぃいぅうぇえぉお... (가나표 전체가 순서대로)
```

이는 **가나표가 순차로 나열된 것** = 16비트 정수값이 1씩 증가하는 바이너리(정점 인덱스·노멀, 텍스처 픽셀, ML 가중치)를 UTF-16으로 잘못 해석한 것일 뿐, 텍스트가 아니다. 실제 일본어 문장/단어는 절대 이렇게 가나표를 통째로 순서대로 늘어놓지 않는다.

---

## 5. 결정적 검증 — 실제 게임 단어로 직접 조회

오탐을 배제하기 위해, **반드시 존재해야 할 실제 일본어 게임 단어**를 직접 바이트 검색했다(UTF-16LE·UTF-8 양쪽).

| 검색어(일본어) | uasset 내 출현 |
|---|---|
| ストラータ (Strata) | **0** |
| カセット (cassette) | **0** |
| 掘削 (굴착) | **0** |
| ステパン (Stepan) | **0** |
| モール (Mole) | **0** |
| 警告 (경고) | **0** |

반면 같은 단어의 **영어 원문**은 다수 존재했다.

| 검색어(영어) | uasset 내 출현 |
|---|---|
| Strata | 158 |
| cassette | 206 |
| Stepan | 18 |
| MOLE | 8 |

→ **게임 소스는 영어이고, 일본어/한국어 텍스트는 오직 `Game.locres`에만 존재한다.** IoStore 어디에도 하드코딩된 일본어 UI 텍스트는 없다.

---

## 6. 결론

| 텍스트 출처 | 상태 |
|---|---|
| `Game.locres` (본문 1300개) | ✅ 전부 한국어 |
| `Engine.locres` 키 표시명 (115개) | ✅ 한국어 ([05 문서](05-engine-keys-and-bugs.md)) |
| IoStore uasset 하드코딩 일본어 | ✅ 0건 (존재하지 않음) |
| `OnlineSubsystem*.locres` (Steam/멀티) | 보류 — 거의 표시 안 되고 대부분 영어 |

**기능적 누락은 없다.** 남은 것은 번역 품질 폴리시(대사의 존댓말/반말, 문맥에 따른 호칭 등) 영역으로, 이는 정적 검증이 아니라 **실제 플레이 검수**로 다듬는다.

> 정리: 정적 분석 도구(`strings`)의 함정과 바이너리 오탐을 모두 걷어내고, "있어야 할 단어가 있는가 / 없어야 할 단어가 없는가"를 양방향으로 확인해 누락 없음을 증명했다.
